# Cosmos DB CMK Configuration - Vault Access Policy example

> This example shows how to deploy the following:
>
> * Cosmos Account
> * Cosmos Database
> * 4 Cosmos containers where each container as specific Index, Throughput and Scaling settings
> * Firewall Settings (allowed subnets, IPs and portal access)
> * Automatic Failover
> * Failover Geo Locations with Priorities
> * Required Tags
> * Several default alerts used with alarm funnel
> * Diagnostic Settings
> * Private Endpoint
> * Customer Managed Keys using Vault Access Policy
